HTML5 details and summary tags

Details and summary tags allow the user to toggle between hiding and showing content without need for Javascript code.

According to CSS Tricks, details and summary tags are "the easiest way ever to make an accordion". Accordions are often used for FAQs because they allow you to display all of the questions to the user while the lengthier answers are hidden from view until the user clicks to open them, one by one.

The default setting for the summary tag provides little black triangles to click to open and close the details, as shown below. However, you can change the default triangles to different looks to match your site's colors or to just change it up using the list-style property. You can see this at work in this CodePen example where the default triangles are switched out for arrows.

It should be noted that the default cursor for the summary tag is NOT the pointer like it is for the link tag. Instead, if you hover over the triangle, by default, you will only see the same arrow that you would see when you hover over non-clickable elements. To clarify the purpose of the summary tag, it may be helpful and more user-friendly to use CSS to change the default cursor to pointer. In the example below, the cursor type was changed to pointer so now clicking on any part of the question opens/closes the answer.

semantic tags

Using semantic tags that explain the role of specific elements on the page helps your site be more accessible. According to MDN Web Docs, semantic writeup improves SEO and optimizes screen readers' ability to support visually impaired users in navigating the site.

Examples of semantic tags are main for the main information of a document, nav for navigation, section for a standalone part of the site, footer and header.

An example of a non-semantic tag is div, as it has no specific role on a page. While divs are great in moderation and can help style the page by chunking elements together, they shouldn't be the only tag on the page since they do not offer any information to people who use screen readers about what the purpose of the given section of the site is.

Below is an example of how a footer might look in html. Spaces were added in the opening and closing tags so that the html code was made visible. In practice, these spaces would be removed. 

< footer>
  This is the footer so it is placed within the footer tag! 
< /footer>

picture tag

The picture tag can be used in place of the img tag. The picture tag accepts more than one image source, allowing you to optimize the image for the user's device resolution.

The example below is from the article Responsive Images with the HTML Picture Tag. It contains additional spaces in the opening and closing tags so that the html is visible. In practice, these spaces would be removed.

< picture >
            < source srcset="small.jpg, small2x.jpg 2x"
              media="(max-width: 639px)">
            < source srcset="medium.jpg, medium2x.jpg 2x"
              media="(min-width: 640px) and (max-width: 1023px)">
            < source srcset="large.jpg, large2x.jpg 2x"
              media="(min-width: 1024px)">
            < img src="my-image.jpg" alt="My image"
              srcset="my-image2x.jpg 2x">

img alt attribute

The alt attribute is an important part of the img tag. Alt text, as in alternative text, makes sure the content of your images are accessible to people who cannot see them. It also helps your site's SEO (search engine optimization) and gives the user context about the image if it does not load, as shown below.

For alt attributes, the more descriptive you can make them, the better for both the user and for SEO. For example, "cat" on its own is not very descriptive. "Cat sleeping" is a little better, but the example belows expand on that even more, allowing the user to get a good idea of exactly what the picture would show them.

CSS custom properties/variables

CSS variables can be set in one css file and reused throughout all of your css files. You can set one color as "--link-color" and reference it as "var(--link-color)". If you ever decide to change up the styles, you'll only have to change it in one place! 

            :root {
            --link-color: #hotpink
            }

            .body__link {
              color: var(--link-color);
            }
MDN Docs: Using CSS Custom Properties

all the CSS background properties

There are so many options for what you can do with the background. CSS has 9 properties for bgs alone. You probably don't use all 9 at a time often, if ever, but the fact that they exist result in endless possibilities. Bg properties enable styling backgrounds so they are responsive to user's devices and therefore enhance the user experience. 

          .container {

text-stroke

text-stroke is not part of CSS3 but is supported by major browsers if you add the prefix -webkit- in front of it. text-stroke is the text effect used for the header of this page. 

            h1 {
              -webkit-text-stroke: 5px orchid;
              color: white;
           }

transitions

CSS3 allows application of animations to elements without needing to use Javascript. This technique was used above with link and :hover. The article 10 CSS3 Properties You Need to Be Familiar With has great examples of transitions as well as other CSS properties! The CSS example below is from the article: 

            ul a {
              border-left: 10px orange solid;
              transition: border-width 0.4s;
           }

          a:hover {
